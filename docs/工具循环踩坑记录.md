# AstraCore 工具循环维护记录

> 只保留容易回归、影响协议兼容或直接影响用户体验的问题。早期实现细节已删除，避免文档变成流水账。

---

## 1. 工具消息必须保持协议完整

**现象**：工具循环中断或达到最大轮次后，Anthropic API 报错 `tool_use ids were found without tool_result blocks`，或报错 `Unexpected role "tool"`。

**根因**：

- Anthropic 要求每个 `tool_use` 都必须有对应的 `tool_result`。
- 内部领域模型可以使用 `MessageRole.TOOL`，但发给 Anthropic 时必须转换成 `role: "user"` 的 `tool_result` content block。
- 如果最后一轮仍允许模型继续调用工具，循环结束时可能留下未执行的 `tool_use`。

**当前策略**：

- 保存历史前用 `_strip_dangling_tool_calls` 清理尾部悬空工具调用。
- 最后一轮工具循环不再传入 tools，强制模型输出文本。
- `AnthropicAdapter._convert_messages` 负责把内部 TOOL 消息转换成 Anthropic 合法格式。

---

## 2. 工具循环必须有最终回答兜底

**现象**：工具调用成功结束，但前端只显示"（空响应）"，用户不知道分析是否完成。

**根因**：工具轮结束时，最后一条消息可能是 `TOOL` 或空 `ASSISTANT`。此时模型还没有机会基于工具结果生成最终总结。

**当前策略**：

- 检测 `_needs_summary_fallback`，必要时触发一次不带 tools 的总结收尾调用。
- 总结前裁剪历史，只保留最近关键工具结果，避免上下文过长导致再次空输出。
- 如果兜底总结仍为空，后端注入续接提示，让用户可以发送"继续"沿用同一 `session_id` 接着分析。

---

## 3. 工具结果必须控制上下文预算

**现象**：一轮工具调用包含多个大结果时，工具结果很快撑爆上下文窗口，导致后续总结失败或空响应。

**根因**：单条工具结果默认过大，多工具并发时 token 量会叠加。例如 9 个工具结果、每条 20 K 字符，单轮就可能超过 100 K 字符。

**当前策略**：在 `config/config.yaml` 的 `agent` 段限制工具结果大小和循环轮次：

```yaml
agent:
  max_tool_result_chars: 6000
  max_tool_iterations: 7
  tool_timeout_s: 60
```

工具需要读取大文件或大目录时，应引导模型使用 `offset/page` 分页读取，不要一次返回完整内容。

---

## 4. 流式文本要区分 thinking 和最终答案

**现象**：模型在工具调用前输出的规划文字被前端当成最终回答展示，用户会误以为回答已经结束。

**根因**：流式路径如果直接把所有 `TEXT_DELTA` 都发成 `message`，就无法区分"工具轮旁白"和"最终答案"。

**当前策略**：

```text
工具轮中遇到 TOOL_CALL -> 当前轮已缓存文本转为 thinking
工具轮后续 TEXT_DELTA -> 继续作为 thinking
最终无工具轮 TEXT_DELTA -> 作为 message 输出
```

前端会过滤空 thinking block，并把工具调用写入 `toolActivity`，让用户知道模型用了哪些工具、是否执行完成。

---

## 5. 历史 thinking block 不能直接重放

**现象**：Claude Sonnet 多轮对话时报 `Invalid signature in thinking block`。

**根因**：Anthropic 的 `thinking` block 带有与当次请求上下文绑定的签名。历史消息重放时上下文已经变化，旧签名会失效。

**当前策略**：

- 重放历史 `anthropic_content_blocks` 时过滤 `thinking` block，只保留 `text` 和 `tool_use`。
- 是否回放 Anthropic 原始 block 由 profile 能力 `anthropic_blocks` 控制。
- DeepSeek Anthropic 兼容接口等代理模型必须谨慎开启 block 回放。

---

## 6. 多模型参数要按能力发送

**现象**：

- DeepSeek 兼容接口不支持 Anthropic 的 `thinking.enabled` 参数。
- Claude Opus 返回 `temperature is deprecated for this model`。

**根因**：不同模型和代理对工具调用、深度思考、temperature、Anthropic block 回放支持不一致，不能用一套固定请求参数覆盖所有模型。

**当前策略**：`model_capabilities.py` 根据 `provider/model/base_url` 自动推导能力：

```yaml
llm:
  default_profile: claude-sonnet
  profiles:
    - id: claude-sonnet
      provider: anthropic
      api_key_env: ANTHROPIC_API_KEY
      model: claude-sonnet-4-6
```

常规配置不需要手写 `capabilities`。只有代理行为与内置表不一致时，才在对应 profile 中局部覆盖。

---

## 配置入口

工具循环相关参数在 `config/config.yaml`：

```yaml
agent:
  max_tool_result_chars: 6000
  max_tool_iterations: 7
  tool_timeout_s: 60
```

默认读取 `config/config.yaml`；如需切换环境，可在 `.env` 设置 `ASTRACORE_CONFIG=config/config.local.yaml`。
