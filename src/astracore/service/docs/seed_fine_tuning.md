# 什么是模型微调（Fine-tuning）

**微调**（Fine-tuning）是在预训练模型的基础上，使用特定领域数据进一步训练，使模型适应专业场景的技术。

## 主要方法

| 方法 | 说明 | 显存需求 |
|------|------|----------|
| **全参数微调** | 更新所有参数，效果最好 | 极高 |
| **LoRA** | 只训练低秩矩阵，参数高效 | 中等 |
| **QLoRA** | LoRA + 量化，显存需求更低 | 低 |

## 微调 vs RAG

| 维度 | 微调 | RAG |
|------|------|-----|
| 知识更新 | 需重新训练 | 更新文档库即可 |
| 推理延迟 | 低（无检索步骤） | 稍高 |
| 数据成本 | 高（需标注数据） | 低 |
| 适用场景 | 风格/格式/能力迁移 | 知识问答/事实检索 |

## 最佳实践

两者并不互斥，**微调 + RAG** 的组合方案在企业场景中效果最佳：用微调适配风格和格式，用 RAG 提供最新知识。
